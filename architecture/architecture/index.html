    Architecture¶

    L'ensemble des codes source de la stack applicative sont hébergés sous l'organisation Avracadabra.

    La pile applicative est composée de 4 éléments: stock, api-gateway, backend et mobile.

    A ne pas manquer le chapitre Assurance qualité

    Stock¶

    Le dépôt stock est le service de gestion de stock a proprement parlé (WMS: Warehouse Management System), développé avec le framework1 AnyBlok et principalement basé sur les modules fournis par AWB: (Anyblok Warehouse management system Base).

    Pourquoi AnyBlok ?

    AnyBlok est un framework plutôt récent (moins de 10 ans) dont j'ai (Pierre Verkest) confiance et contribue ! Il repose sur des librairie connue tel que SqlAlchemy, pyramid.

    API Gateway¶

    Le dépôt api-gateway est le code source du service passerelle (ou proxy) jouant un rôle d'intermédiaire entre les interfaces utilisateurs (backend et mobile) et les différents services métiers.

    Services métiers¶

    L'api gateway permet d'agréger des données provenant de plusieurs services tels que le service stock, service de gestion de produit (PIM: Product Information Management)...

    Merci ascynio qui facilite la parallélisation des appels aux différentes API métiers grace notamment à la lib httpx.

    Aspects techniques¶

    Cette passerelle expose les données au travers d'API GraphQL, dans l'objectif de découpler, un minimum, le code source des interfaces utilisateurs et des services métiers.

    Le moteur GraphQL tartiflette est utilisé car il est prévu pour fonctionner naturellement avec asyncio.

    Le service est donc totalement conçu pour fonctionner en async/io avec notamment:

    • starlette: Un framework ASGI
    • tartiflette-asgi qui fait la "glue" entre starlette et tartiflette.
    • uvicorn le serveur ASGI.
    • Gunicorn qui permet de gérer les processus (ASGI) de manière résiliente.

    backend¶

    Le dépôt backend est une application VueJs dont les cas d'usage sont pensé pour PC ou tablette et uniquement testé sur les navigateurs PC.

    mobile¶

    Le dépôt mobile est une application mobile, outil asynchrone prévu pour gérer les déconnexions réseau (mode offline) permettant de travailler dans des locaux ou le reseaux serait faible avec un smartphone ou un scanner sous Android.

    Assurance qualité¶

    Le dépôt QA contient:

    • Les test d'intégration sous la forme de Programmation pilotée par le comportement (BDD: Behavior-driven development) qui doit permettre à un non technique de comprendre les jeux de test.

    1. un framework est une boite à outil pour développeur. Il fournis des primitives facilement réutilisable. ↩
